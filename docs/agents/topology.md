# Docs Topology

## Root-level docs
- `README.md`: human-facing project overview
- `AGENTS.md`: short operational guide for coding agents

## Agent docs
- `docs/agents/*.md`: deeper operational details, workflow notes, roadmap, verification

## Principle
- Keep durable commands and rules near the top.
- Move detailed rationale and evolving plans into deeper docs.
- Prefer scripts/tests/CI over prose for enforceable behavior.
